Web Architecture

Client-Server Model

The web operates on a client-server model. A client (usually a browser) sends requests to a server, which processes the request and sends back a response.

[ Browser (Client) ] ——HTTP Request——→ [ Web Server ]
[ Browser (Client) ] ←——HTTP Response—— [ Web Server ]
                                        ↕
                                [ Database Server ]

Three-Tier Architecture

🖥️

Presentation Tier (Front-end)

What the user sees and interacts with. Built with HTML, CSS, and JavaScript running in the browser.

⚙️

Application Tier (Back-end)

Business logic and processing. Handles requests from clients, processes data, and communicates with the database. Built with Python, Node.js, PHP, Java, etc.

🗄️

Data Tier (Database)

Stores and retrieves data. Relational (MySQL, PostgreSQL) or NoSQL (MongoDB). Only the application tier should communicate directly with the database.

DNS — Domain Name System

DNS translates human-readable domain names (e.g., www.example.com) into IP addresses. It is the "phonebook of the internet".

IP Addresses and Ports

  • An IP address identifies a device on a network (e.g., 192.168.1.1 for IPv4, or 2001:db8::1 for IPv6)
  • A port identifies a specific service on that device. HTTP uses port 80; HTTPS uses port 443; SSH uses port 22.
  • A URL such as https://example.com:443/path combines protocol, domain, port, and path.

HTTP Protocol

Hypertext Transfer Protocol (HTTP) is the foundation of data communication on the web. HTTPS is HTTP secured with TLS encryption.

HTTP Methods (Verbs)

MethodActionSafe?Idempotent?Example Use
GETRetrieve a resource✅✅Fetch a webpage or API data
POSTCreate a new resource❌❌Submit a form, create a user
PUTReplace a resource entirely❌✅Update a user profile completely
PATCHPartially update a resource❌✅Update only a user's email

ℹ️
Safe vs Idempotent

Safe: does not modify server state. Idempotent: same request made multiple times produces the same result as one request.

JavaScript

JavaScript is the programming language of the web. It runs in the browser and enables dynamic, interactive web pages.

DOM Manipulation

The Document Object Model (DOM) represents the HTML page as a tree of objects that JavaScript can read and modify.

JavaScript — DOM Manipulation 
// Select elements
const btn = document.querySelector('#myButton');
const items = document.querySelectorAll('.item');

// Modify content
document.querySelector('h1').textContent = 'Hello World';
document.querySelector('.card').classList.add('active');

// Create and append elements
const li = document.createElement('li');
li.textContent = 'New Item';
document.querySelector('ul').appendChild(li);

// Event listeners
btn.addEventListener('click', function(event) {
  alert('Button clicked!');
});

Fetch API — Making HTTP Requests

JavaScript — Fetch API with async/await 
// GET request
async function getUser(id) {
  try {
    const response = await fetch(`/api/users/${id}`);

    if (!response.ok) {
      throw new Error(`HTTP error: ${response.status}`);
    }

    const data = await response.json();
    console.log(data); // { id: 1, name: "Alice" }
    return data;
  } catch (error) {
    console.error('Failed to fetch user:', error);
  }
}

// POST request
async function createUser(userData) {
  const response = await fetch('/api/users', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(userData)
  });
  return response.json();
}

Promises and Async/Await

JavaScript is single-threaded and non-blocking. Asynchronous operations (like network requests) use Promises to handle eventual results without blocking execution.

ConceptDescription
SynchronousCode executes line by line; each line waits for the previous
AsynchronousOperations that take time are scheduled; code continues executing
PromiseAn object representing the eventual result of an async operation (pending → fulfilled/rejected)
async/awaitSyntactic sugar over Promises for cleaner asynchronous code
CallbackOlder pattern: a function passed to an async function to be called when done

Rendering Strategies

🖥️

Server-Side Rendering (SSR)

HTML is generated on the server and sent to the client. Good for SEO and initial load. Examples: Django templates, PHP, Next.js (SSR mode).

⚡

Client-Side Rendering (CSR)

Browser downloads a minimal HTML shell and a JavaScript bundle. JS renders the UI. Faster interactions after initial load. Examples: React SPA, Vue SPA.

🔀

Hybrid / Static Site Generation

Pages are pre-rendered at build time. Fast and secure. Best for content that doesn't change often. Examples: Next.js, Gatsby, Hugo.

APIs & REST

API — Application Programming Interface
A defined contract that allows two software systems to communicate. A web API allows a client to interact with a server by making HTTP requests to defined endpoints.

REST — Representational State Transfer

REST is an architectural style (not a protocol) for designing networked APIs. A RESTful API uses standard HTTP methods and treats everything as a resource with a URL.

Stateless — each request contains all information needed; no session state on the server
Resource-based — everything is a resource identified by a URI
Standard HTTP methods — GET, POST, PUT, PATCH, DELETE
JSON or XML — data transferred in a standard format
Uniform interface — consistent, predictable structure
REST API Design — User Resource 
GET    /api/users          → List all users
POST   /api/users          → Create a new user
GET    /api/users/{id}     → Get a specific user
PUT    /api/users/{id}     → Replace a user
PATCH  /api/users/{id}     → Update fields of a user
DELETE /api/users/{id}     → Delete a user

GET    /api/users/{id}/posts  → List posts by a user

JSON — JavaScript Object Notation

JSON is the most common data format for web APIs. It is lightweight, human-readable, and language-independent.

Python Flask REST API Example

Python / Flask — Simple REST API 
from flask import Flask, jsonify, request

app = Flask(__name__)

users = [
    {"id": 1, "name": "Alice"},
    {"id": 2, "name": "Bob"}
]

@app.route('/api/users', methods=['GET'])
def get_users():
    return jsonify(users)

@app.route('/api/users/<int:user_id>', methods=['GET'])
def get_user(user_id):
    user = next((u for u in users if u["id"] == user_id), None)
    if user:
        return jsonify(user)
    return jsonify({"error": "Not found"}), 404

@app.route('/api/users', methods=['POST'])
def create_user():
    data = request.get_json()
    new_user = {"id": len(users) + 1, "name": data["name"]}
    users.append(new_user)
    return jsonify(new_user), 201

Basic SQL

SQL — Core Operations 
-- CREATE a table
CREATE TABLE users (
    id       INTEGER PRIMARY KEY AUTOINCREMENT,
    name     TEXT NOT NULL,
    email    TEXT UNIQUE NOT NULL,
    created  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- INSERT data
INSERT INTO users (name, email)
VALUES ('Alice', 'alice@example.com');

-- SELECT with filtering
SELECT name, email FROM users
WHERE id > 10
ORDER BY name ASC;

-- UPDATE
UPDATE users SET email = 'newemail@example.com'
WHERE id = 1;

-- DELETE
DELETE FROM users WHERE id = 5;

-- JOIN two tables
SELECT users.name, posts.title
FROM posts
INNER JOIN users ON posts.user_id = users.id;

Web Security

Common Web Vulnerabilities

🚨
SQL Injection (SQLi)

Malicious SQL is injected into input fields and executed by the database. Prevention: always use parameterised queries/prepared statements. Never concatenate user input into SQL.

🚨
Cross-Site Scripting (XSS)

Attacker injects malicious JavaScript into a page viewed by other users, potentially stealing cookies or redirecting to phishing pages. Prevention: encode output; use Content Security Policy (CSP) headers; never use innerHTML with user data.

🚨
Cross-Site Request Forgery (CSRF)

Tricks authenticated users into submitting requests they didn't intend (e.g., transferring funds). Prevention: use CSRF tokens; same-site cookie attribute; verify Origin and Referer headers.

Security Headers

HeaderPurpose
Strict-Transport-SecurityForces HTTPS for all connections (HSTS)
Content-Security-PolicyRestricts sources of scripts, styles, images (prevents XSS)
X-Content-Type-Options: nosniffPrevents MIME type sniffing
X-Frame-Options: DENYPrevents clickjacking via iframes
Referrer-PolicyControls how much referrer info is shared
✅
Web Programming Key Takeaways
  • HTTP is stateless; HTTPS adds encryption via TLS
  • REST APIs use HTTP methods to CRUD resources
  • Always validate input server-side; never trust client-side validation alone
  • Use parameterised queries to prevent SQL injection
  • Encode all output to prevent XSS
  • JSON is the standard data format for modern web APIs
