Problem Solving & Algorithm Design

Algorithm
A finite, ordered set of instructions for solving a well-defined problem. An algorithm must be unambiguous, finite, and produce a correct result.

Problem-Solving Steps

  1. 1

    Understand the Problem

    Read carefully. Identify inputs, outputs, and constraints. What are the edge cases?

  2. 2

    Design an Algorithm

    Use pseudocode or a flowchart to describe the solution step-by-step before writing code.

  3. 3

    Implement in Code

    Translate the algorithm into a programming language. Follow coding conventions.

  4. 4

    Test & Debug

    Test with various inputs including edge cases. Fix any errors.

  5. 5

    Evaluate

    Is the solution correct? Is it efficient? Is it readable and maintainable?

Pseudocode

Pseudocode is an informal, high-level description of an algorithm using plain English-like statements. It focuses on logic, not syntax.

Pseudocode — Find the Largest Number 
INPUT: list of numbers
OUTPUT: the largest number

largest ← first number in list
FOR EACH number IN list:
    IF number > largest THEN
        largest ← number
OUTPUT largest

Flowcharts

Flowcharts use standard symbols to represent algorithms visually:

Oval — Start / End (terminal)
Rectangle — Process / Action
Diamond — Decision (yes/no question)
Parallelogram — Input / Output
Arrow — Flow of control
Cylinder — Data storage

Variable Naming Rules

  • Must start with a letter or underscore
  • Can contain letters, digits, and underscores
  • Case-sensitive (age ≠ Age)
  • Cannot use reserved keywords (if, for, class, etc.)
  • Use snake_case for Python variables: student_name
  • Use descriptive names: total_marks not tm

Control Structures

Control structures determine the order of execution of instructions in a program.

Selection — if/elif/else

Python — Selection 
score = 72

if score >= 90:
    grade = 'A'
elif score >= 80:
    grade = 'B'
elif score >= 70:
    grade = 'C'

Iteration — while loop

Python — while loop

# while True with break — sentinel loop

    user_input = input("Enter number (or 'quit'): ")
    if user_input == 'quit':

Functions & Procedures

⚙️

Function

A named block of code that performs a task and returns a value. Called to compute a result.

🔧

Procedure

A named block of code that performs a task but does not return a value. Called for its side effects (e.g., printing, modifying state).

Python — Functions 
# Function with parameters and return value
def calculate_area(length, width):
    """Return the area of a rectangle."""
    area = length * width
    return area

result = calculate_area(5, 3)  # result = 15

# Default parameters
def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

print(greet("Alice"))          # Hello, Alice!
print(greet("Bob", "Hi"))    # Hi, Bob!

# Multiple return values
def min_max(numbers):
    return min(numbers), max(numbers)

smallest, largest = min_max([4, 1, 7, 2])  # 1, 7

Bubble Sort

Repeatedly steps through the list, compares adjacent elements, and swaps them if out of order. The largest element "bubbles" to the end each pass.

Python — Bubble Sort 
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:
            break  # Already sorted — optimisation
    return arr

numbers = [64, 34, 25, 12, 22, 11, 90]
print(bubble_sort(numbers))  # [11, 12, 22, 25, 34, 64, 90]

Selection Sort

Finds the minimum element from the unsorted part and places it at the beginning. Performs n-1 passes, selecting the minimum each time.

Python — Selection Sort 
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_index = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_index]:
                min_index = j
        arr[i], arr[min_index] = arr[min_index], arr[i]
    return arr

Insertion Sort

Builds a sorted portion from the left. Each new element is inserted into its correct position in the already-sorted portion. Efficient for small or nearly-sorted lists.

Python — Insertion Sort 
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr

Searching Algorithms

Linear Search

Checks each element one by one from the beginning. Works on any list (sorted or unsorted). Time complexity: O(n) worst case.

Python — Linear Search 
def linear_search(arr, target):
    for index, value in enumerate(arr):
        if value == target:
            return index  # found at this index
    return -1         # not found

Binary Search

Requires a sorted list. Repeatedly halves the search space. Much faster than linear search for large sorted lists. Time complexity: O(log n).

Python — Binary Search 
def binary_search(arr, target):
    left, right = 0, len(arr) - 1

    while left <= right:
        mid = (left + right) // 2

        if arr[mid] == target:
            return mid         # found!
        elif arr[mid] < target:
            left = mid + 1    # target is in right half
        else:
            right = mid - 1   # target is in left half

    return -1  # not found

sorted_list = [11, 12, 22, 25, 34, 64, 90]
print(binary_search(sorted_list, 25))  # 3 (index)

Linear SearchNoO(n)Small or unsorted lists
Binary SearchYesO(log n)Large sorted lists

Recursion & File I/O

Recursion

Recursion
A function that calls itself to solve a smaller version of the same problem. Every recursive function must have a base case (stopping condition) to prevent infinite recursion.
Python — Recursive Factorial 
def factorial(n):
    # Base case: factorial of 0 or 1 is 1
    if n <= 1:
        return 1
    # Recursive case: n! = n × (n-1)!
    return n * factorial(n - 1)

# factorial(5) → 5 × 4 × 3 × 2 × 1 = 120
print(factorial(5))  # 120
⚠️
Always Define a Base Case

Without a base case, a recursive function will call itself indefinitely until Python raises a RecursionError: maximum recursion depth exceeded.

File I/O

Python — Reading and Writing Files 
# Writing to a file
with open("data.txt", "w") as file:
    file.write("Line 1\n")
    file.write("Line 2\n")

# Reading entire file
with open("data.txt", "r") as file:
    content = file.read()
    print(content)

# Reading line by line
with open("data.txt", "r") as file:
    for line in file:
        print(line.strip())  # strip() removes trailing newline

# Appending to a file
with open("data.txt", "a") as file:
    file.write("Line 3\n")
ℹ️
File Modes
  • "r" — read (file must exist)
  • "w" — write (creates or overwrites)
  • "a" — append (adds to end of file)
  • "x" — exclusive create (fails if file exists)
  • Add "b" for binary mode: "rb", "wb"
✅
Programming Fundamentals Key Takeaways
  • Design algorithms with pseudocode before writing code
  • Use for loops for known iterations; while for condition-based loops
  • Functions promote reuse and modularity; always define a docstring
  • Understand list indexing (arr[0] is first, arr[-1] is last)
  • Binary search is O(log n) but requires a sorted list; linear search is O(n)
  • Recursion requires a base case to prevent infinite loops
  • Always use with open(...) for safe file handling
