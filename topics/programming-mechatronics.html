What is Mechatronics?

Mechatronics
A multidisciplinary engineering field that combines mechanical engineering, electrical engineering, computer science, and control systems to design and create intelligent systems and products.

Components of a Mechatronic System

🔌

Sensors (Input)

Detect physical properties from the environment (temperature, light, distance, motion) and convert them to electrical signals for the controller.

🧠

Microcontroller (Processing)

The "brain" of the system. Receives sensor data, runs the program, and sends commands to actuators.

⚡

Actuators (Output)

Convert electrical signals back into physical actions (movement, light, sound). Examples: motors, LEDs, speakers, servos.

🔄

Feedback Loop

Sensors monitor the output of actuators, feeding information back to the controller to adjust behaviour. This is the basis of a closed-loop control system.

Open-Loop vs Closed-Loop Control

TypeDescriptionFeedback?Example
Open-Loop Controller sends a fixed command; no monitoring of outcome ❌ No A toaster — set timer, doesn't check if toast is done
Closed-Loop Controller monitors output and adjusts commands accordingly ✅ Yes Air conditioning — checks room temperature and adjusts

Examples of Mechatronic Systems

Automatic door — PIR sensor + motor
Robot arm — servo motors + encoders + controller
3D printer — stepper motors + limit switches + thermistor
Smart irrigation — soil moisture sensor + water pump + timer
Line-following robot — IR sensors + DC motors
Anti-lock braking system (ABS) — wheel speed sensors + braking actuators

Arduino Platform

Arduino is the most popular platform for learning embedded programming. It consists of:

  • Hardware: Arduino board (e.g., Uno, Mega, Nano) with I/O pins
  • Software: Arduino IDE + C/C++-based programming language
  • Ecosystem: Large library of code for common sensors and devices

Arduino Uno Key Specifications

Microcontroller: ATmega328P (8-bit)
Clock speed: 16 MHz
Flash memory: 32 KB (program storage)
SRAM: 2 KB (runtime variables)
Digital I/O pins: 14 (6 with PWM)
Analog input pins: 6 (10-bit ADC)
Operating voltage: 5V

Digital vs Analog Signals

💡

Digital Signal

Has only two discrete states: HIGH (1) and LOW (0). On Arduino: HIGH = 5V, LOW = 0V. Clean, noise-resistant. Examples: button pressed/not pressed, LED on/off.

〰️

Analog Signal

Continuous range of values. Can be any value within a range (e.g., 0–5V). Represents real-world quantities like temperature or light intensity. Examples: potentiometer, temperature sensor output.

ADC — Analog-to-Digital Conversion

Microcontrollers can only process digital data. An Analog-to-Digital Converter (ADC) converts analog voltages to digital numbers.

  • Arduino Uno has a 10-bit ADC: converts 0–5V to values 0–1023
  • Resolution = 5V / 1024 ≈ 4.9 mV per step
  • Formula: voltage = (analog_value / 1023) × 5.0
Arduino C++ — Reading an Analog Sensor 
const int SENSOR_PIN = A0;  // Analog pin A0

void setup() {
    Serial.begin(9600);  // Start serial communication
}

void loop() {
    int raw_value = analogRead(SENSOR_PIN);  // Read 0-1023
    float voltage = (raw_value / 1023.0) * 5.0;  // Convert to volts

    Serial.print("Raw: ");
    Serial.print(raw_value);
    Serial.print(" | Voltage: ");
    Serial.print(voltage, 2);
    Serial.println("V");

    delay(500);  // Wait 500ms
}

PWM — Pulse Width Modulation

PWM (Pulse Width Modulation)
A technique to simulate an analog output using a digital pin by rapidly switching it ON and OFF. The duty cycle (percentage of time the signal is HIGH) controls the effective voltage. Higher duty cycle = more power delivered.

100% duty cycle: ██████████ (full brightness/speed)
50% duty cycle: █░░░░█░░░░ (half brightness/speed)
25% duty cycle: █░░░░░░░░░ (quarter brightness)

Arduino C++ — PWM to fade an LED 
const int LED_PIN = 9;  // Must be a PWM pin (~)

void setup() {
    pinMode(LED_PIN, OUTPUT);
}

void loop() {
    // Fade in
    for (int brightness = 0; brightness <= 255; brightness++) {
        analogWrite(LED_PIN, brightness);  // 0-255 maps to 0-5V
        delay(10);
    }
    // Fade out
    for (int brightness = 255; brightness >= 0; brightness--) {
        analogWrite(LED_PIN, brightness);
        delay(10);
    }
}

Ultrasonic Distance Sensor (HC-SR04)

The HC-SR04 measures distance by sending an ultrasonic pulse and measuring the time for the echo to return.

Distance (cm) = (Time of echo in μs) / 58

Or: Distance = (Speed of sound × Time) / 2
Speed of sound ≈ 343 m/s at 20°C

Arduino C++ — Ultrasonic Distance Sensor 
const int TRIG_PIN = 9;
const int ECHO_PIN = 10;

void setup() {
    Serial.begin(9600);
    pinMode(TRIG_PIN, OUTPUT);
    pinMode(ECHO_PIN, INPUT);
}

void loop() {
    // Send trigger pulse (10μs HIGH)
    digitalWrite(TRIG_PIN, LOW);
    delayMicroseconds(2);
    digitalWrite(TRIG_PIN, HIGH);
    delayMicroseconds(10);
    digitalWrite(TRIG_PIN, LOW);

    // Measure echo duration
    long duration = pulseIn(ECHO_PIN, HIGH);

    // Calculate distance in cm
    float distance = duration / 58.0;

    Serial.print("Distance: ");
    Serial.print(distance, 1);
    Serial.println(" cm");

    delay(200);
}

Servo Motor Control

A servo motor can be rotated to a precise angle between 0° and 180°. It uses a PWM signal with a specific pulse width to control position.

Arduino C++ — Servo Motor 
#include <Servo.h>

Servo myServo;

void setup() {
    myServo.attach(9);  // Servo connected to pin 9
}

void loop() {
    // Sweep from 0° to 180°
    for (int angle = 0; angle <= 180; angle += 5) {
        myServo.write(angle);
        delay(50);
    }
    // Sweep back from 180° to 0°
    for (int angle = 180; angle >= 0; angle -= 5) {
        myServo.write(angle);
        delay(50);
    }
}

Motor Drivers

DC motors require more current than an Arduino pin can supply (max 40mA per pin). A motor driver IC (such as the L298N or L293D) acts as a power interface between the microcontroller and the motor. It also allows direction control (H-bridge circuit).

Arduino Programming

Arduino sketches (programs) are written in a C/C++ dialect. Every sketch has two required functions:

🔧

setup()

Runs once when the Arduino powers on or resets. Used to initialise pins, start serial communication, configure libraries.

🔄

loop()

Runs repeatedly in an infinite loop after setup() completes. Contains the main program logic.

Digital I/O

Arduino C++ — Digital I/O: Button + LED 
const int LED_PIN    = 13;
const int BUTTON_PIN = 2;

void setup() {
    pinMode(LED_PIN, OUTPUT);
    pinMode(BUTTON_PIN, INPUT_PULLUP);  // use internal pull-up resistor
    Serial.begin(9600);
}

void loop() {
    int buttonState = digitalRead(BUTTON_PIN);

    // INPUT_PULLUP: LOW when pressed, HIGH when released
    if (buttonState == LOW) {
        digitalWrite(LED_PIN, HIGH);  // LED on
        Serial.println("Button pressed");
    } else {
        digitalWrite(LED_PIN, LOW);   // LED off
    }
}

Traffic Light Project Example

Arduino C++ — Traffic Light Sequence 
const int RED_PIN    = 11;
const int YELLOW_PIN = 10;
const int GREEN_PIN  = 9;

void setLight(int red, int yellow, int green) {
    digitalWrite(RED_PIN, red);
    digitalWrite(YELLOW_PIN, yellow);
    digitalWrite(GREEN_PIN, green);
}

void setup() {
    pinMode(RED_PIN, OUTPUT);
    pinMode(YELLOW_PIN, OUTPUT);
    pinMode(GREEN_PIN, OUTPUT);
}

void loop() {
    setLight(HIGH, LOW, LOW);   // Red   — stop
    delay(5000);

    setLight(HIGH, HIGH, LOW);  // Red+Yellow — prepare
    delay(1000);

    setLight(LOW, LOW, HIGH);   // Green — go
    delay(4000);

    setLight(LOW, HIGH, LOW);   // Yellow — slow down
    delay(1500);
}

Communication Protocols

Communication protocols define how devices exchange data. Different protocols have different trade-offs in speed, complexity, and the number of wires required.

ProtocolTypeWiresSpeedUse Case
UART (Serial) Asynchronous 2 (TX, RX) Up to 115200 baud Arduino Serial monitor, GPS, Bluetooth modules
I2C Synchronous, multi-device 2 (SDA, SCL) 100 kHz – 400 kHz Multiple sensors on same bus (OLED, IMU, RTC)
SPI Synchronous, high-speed 4 (MOSI, MISO, SCK, SS) Up to MHz SD cards, LCD displays, fast ADCs
1-Wire Synchronous 1 (data + power) Low Temperature sensors (DS18B20)

I2C Protocol Detail

I2C uses just two wires to communicate with multiple devices. Each device has a unique 7-bit address, allowing up to 128 devices on the same bus.

SDA — Serial Data line (bidirectional)
SCL — Serial Clock line (controlled by master)
Master — initiates and controls communication (usually the microcontroller)
Slave — responds to master requests (sensors, displays)
Pull-up resistors — required on both SDA and SCL lines
ℹ️
Finding I2C Device Addresses

If you're unsure of a device's I2C address, upload an I2C Scanner sketch to your Arduino. It scans all addresses (0x03 to 0x77) and reports which ones respond.

Embedded Systems & IoT

Embedded Systems

Embedded System
A computer system with a dedicated function within a larger mechanical or electrical system. It is "embedded" as part of a complete device. Optimised for specific tasks: low power, small size, real-time constraints.

Real-Time Systems

Many embedded systems must respond to events within strict time constraints:

⚡

Hard Real-Time

Missing a deadline causes system failure or danger. Example: airbag control system must deploy within milliseconds of collision detection.

🕐

Soft Real-Time

Missing a deadline degrades performance but doesn't cause failure. Example: video streaming — late frames reduce quality but don't crash the system.

Internet of Things (IoT)

IoT — Internet of Things
A network of physical devices ("things") embedded with sensors, software, and connectivity that enables them to collect and exchange data over the internet.
ESP8266 / ESP32 — Wi-Fi enabled microcontrollers popular for IoT projects
MQTT — lightweight messaging protocol designed for IoT (publish/subscribe model)
Cloud platforms — AWS IoT, Google Cloud IoT, ThingSpeak for data logging
Edge computing — processing data locally on the device rather than sending to the cloud

IoT Security Concerns

⚠️
IoT Security Risks
  • Default credentials not changed (e.g., admin/admin)
  • Unencrypted data transmission over public networks
  • Insecure firmware with vulnerabilities
  • Large attack surface (millions of connected devices)
  • Difficulty patching/updating in the field

Programming with MicroPython

MicroPython is a lean implementation of Python 3 designed to run on microcontrollers. It's easier to read than Arduino's C++, making it great for learning.

MicroPython — Blink LED (Raspberry Pi Pico) 
from machine import Pin
import utime

# Built-in LED on Raspberry Pi Pico is pin 25
led = Pin(25, Pin.OUT)

while True:
    led.high()         # Turn LED on
    utime.sleep(0.5)  # Wait 500ms
    led.low()          # Turn LED off
    utime.sleep(0.5)
✅
Mechatronics Key Takeaways
  • A mechatronic system has: Sensors (input) → Microcontroller (process) → Actuators (output)
  • Closed-loop systems use feedback to correct output; open-loop do not
  • Digital signals are HIGH/LOW; analog signals are continuous — use ADC to convert
  • PWM simulates analog output by varying the duty cycle
  • Arduino's setup() runs once; loop() runs continuously
  • Use motor drivers (L298N) to control DC motors — never connect directly to Arduino pins
  • I2C connects many devices with just 2 wires; each device has a unique address
  • IoT devices must be secured: change defaults, encrypt communications, patch firmware
