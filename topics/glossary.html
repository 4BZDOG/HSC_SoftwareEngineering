A

Abstraction

📐 Programming Fundamentals 🧩 OOP

The concept of hiding complex internal implementation details while exposing only the necessary interface to the user or calling code. Reduces cognitive complexity and increases maintainability.

See also: Encapsulation

Agile Methodology

🔄 SDLC 📋 SE Project

An iterative, incremental approach to software development. Agile prioritises working software, customer collaboration, and responsiveness to change over rigid, upfront specification.

See also: Scrum, WAGILE

API (Application Programming Interface)

🌐 Programming for the Web ⚙️ Software Automation

A contract defining how software components communicate. An API specifies what requests can be made, what data formats to use, and what responses to expect — without exposing implementation details.

See also: REST

Algorithm

📐 Programming Fundamentals 🔄 SDLC

A step-by-step procedure required to solve a problem or accomplish a task. Algorithms are defined by their efficiency, clarity, and finiteness.

See also: Pseudocode, Flowchart

Array

📐 Programming Fundamentals

A data structure consisting of a collection of elements, each identified by at least one index or key. Elements are usually stored in contiguous memory locations.

↑ Back to Top
B

Backlog (Product Backlog)

🔄 SDLC 📋 SE Project

A prioritised list of features, user stories, bug fixes, and technical tasks yet to be completed. In Scrum, the Product Backlog is the single source of work for the development team.

Black-Box Testing

🔄 SDLC ⚙️ Software Automation

A testing technique that validates software behaviour from an external perspective, without knowledge of internal code. Testers supply inputs and verify outputs against expected results.

See also: White-Box Testing

Binary Search

📐 Programming Fundamentals

An efficient search algorithm that finds an item from a sorted list by repeatedly dividing the search interval in half. It has a time complexity of O(log n).

See also: Linear Search

Beta Testing

🔄 SDLC

The second phase of software testing in which a sampling of the intended audience tries the product out. It follows alpha testing and focuses on real-world usage and edge cases.

↑ Back to Top
C

CI/CD (Continuous Integration / Continuous Deployment)

⚙️ Software Automation

A DevOps practice where code changes are automatically built, tested (CI) and deployed to staging or production environments (CD). Enables rapid, reliable delivery cycles with minimal manual intervention.

Class

🧩 OOP

A blueprint or template defining the structure (attributes) and behaviour (methods) for a family of objects. Instantiation creates an individual Object from the class definition in memory.

See also: Object, Inheritance

Cohesion

📐 Programming Fundamentals 🧩 OOP

A measure of how closely related the responsibilities of a single module or class are. High cohesion (single-responsibility) is a hallmark of good design — each unit does one thing well.

See also: Coupling

Coupling

🧩 OOP 🔒 Secure Architecture

The degree of interdependence between software modules. Loose coupling — minimising direct knowledge between modules — improves maintainability, testability, and security isolation.

See also: Cohesion
↑ Back to Top
D

DAST (Dynamic Application Security Testing)

🔒 Secure Architecture ⚙️ Software Automation

Security testing performed against a running application from the outside — simulating the perspective of an attacker. Identifies vulnerabilities such as SQL injection and XSS at runtime.

See also: SAST

DOM (Document Object Model)

🌐 Programming for the Web

A tree-structured programming interface representing an HTML or XML document. JavaScript can traverse and manipulate the DOM to dynamically update page content, structure, and style without a full page reload.

Data Dictionary

🔄 SDLC 📋 SE Project

A centralized repository of information about data, defining its properties, relationships, origin, usage, and format. Essential for the Design and Implementation phases of the SDLC.

Debugging

📐 Programming Fundamentals ⚙️ Software Automation

The systematic process of finding and resolving bugs or defects within software. Involves isolating the cause, developing a fix, and verifying the resolution.

See also: Unit Testing
↑ Back to Top
E

Encapsulation

🧩 OOP

The bundling of data (attributes) and the methods operating on that data within a single unit (class). Access modifiers (public, private, protected) control visibility, preventing unintended external interference.

See also: Abstraction
↑ Back to Top
F

Finite State Machine (FSM)

📐 Programming Fundamentals 🤖 Mechatronics

A computational model with a finite set of states, defined transitions triggered by inputs or events, and associated actions. FSMs are fundamental to modelling embedded systems, protocols, and UI logic.

Flowchart

📐 Programming Fundamentals 🔄 SDLC

A diagrammatic representation of an algorithm, showing the steps as boxes of various kinds, and their order by connecting them with arrows. NESA requires standard symbols for Begin/End, Decision, and Input/Output.

See also: Algorithm, Pseudocode

Functional Requirements

🔄 SDLC 📋 SE Project

The specific behaviors, features, and functions that a system must provide. They describe *what* the system should do, as opposed to non-functional requirements which describe *how* it should perform.

↑ Back to Top
G

Gantt Chart

🔄 SDLC 📋 SE Project

A horizontal bar chart used in project planning to visualise task schedules, durations, and dependencies over time. A key tool during the Planning phase of the SDLC.

GUI (Graphical User Interface)

🌐 Programming for the Web 🔄 SDLC

An interface allowing users to interact with software through visual elements (buttons, menus, icons) rather than text commands. GUI design is evaluated against usability heuristics during testing.

↑ Back to Top
H

Headless CMS

🌐 Programming for the Web

A back-end-only content management system that stores and delivers content via API, decoupled from any specific front-end presentation layer. Allows content to be rendered on any platform or framework.

↑ Back to Top
I

Inheritance

🧩 OOP

A mechanism where a subclass (child) automatically acquires the attributes and methods of a superclass (parent). Enables code reuse and establishes hierarchical "is-a" relationships between types.

See also: Polymorphism, Class

Iteration

📐 Programming Fundamentals 🔄 SDLC

The repetition of a process in order to generate a sequence of outcomes. In programming, this refers to loops (for, while); in management, it refers to repeated development cycles in Agile/Scrum.

See also: Recursion, Agile

IDE (Integrated Development Environment)

📐 Programming Fundamentals ⚙️ Software Automation

A software application that provides comprehensive facilities to computer programmers for development, typically including a source code editor, build automation tools, and a debugger.

↑ Back to Top
J

JWT (JSON Web Token)

🔒 Secure Architecture 🌐 Programming for the Web

An open standard (RFC 7519) for securely transmitting claims between parties as a compact, URL-safe JSON object. Widely used in stateless authentication flows — the server issues a signed token the client presents on each request.

↑ Back to Top
L

Linear Search

📐 Programming Fundamentals

A simple search algorithm that checks every element in a list sequentially until a match is found or the end of the list is reached. Time complexity is O(n).

See also: Binary Search

Logic Error

📐 Programming Fundamentals 🔄 SDLC

An error in a program that causes it to operate incorrectly but not to crash. The code is syntactically correct, but the output is not what the programmer intended.

See also: Syntax Error, Debugging
↑ Back to Top
M

Mechatronics

🤖 Mechatronics

An interdisciplinary field integrating mechanical engineering, electrical engineering, and software to design and operate intelligent systems, robotics, and embedded devices.

MVC (Model-View-Controller)

🌐 Programming for the Web

An architectural design pattern separating an application into three layers: Model (data & business logic), View (UI presentation), and Controller (handles user input). Improves maintainability and separation of concerns.

↑ Back to Top
O

Object

🧩 OOP

A specific runtime instance of a class, allocated in memory with its own state (attribute values). Multiple objects can exist from the same class blueprint simultaneously, each maintaining independent state.

See also: Class
↑ Back to Top
P

Polymorphism

🧩 OOP

The ability for objects of different types to respond to the same method call — each in their own type-specific way. Achieved through method overriding (runtime polymorphism) or method overloading (compile-time).

See also: Inheritance

Pseudocode

📐 Programming Fundamentals 🔄 SDLC

An informal, high-level description of an algorithm using structured plain English. Used in the Design phase of the SDLC to communicate logic without language-specific syntax. NESA specifies a standard pseudocode format for HSC examinations.

Prototype

🔄 SDLC 📋 SE Project

An early sample or model built to test a concept or process. Prototypes help stakeholders visualize the final system and elicit more accurate requirements early on.

Peer Review

🔄 SDLC 📋 SE Project

A process where colleagues examine a software product (code, documentation) to find and fix defects before it moves to the next stage. Improves quality and knowledge sharing.

↑ Back to Top
R

REST (Representational State Transfer)

🌐 Programming for the Web

An architectural style for designing networked APIs that use standard HTTP methods (GET, POST, PUT, DELETE) and stateless communication. RESTful APIs are the dominant approach for modern web services.

See also: API

Recursion

📐 Programming Fundamentals

A programming technique where a function calls itself to solve a problem. Each call reduces the problem to a smaller sub-problem until a base case is reached.

See also: Iteration

Requirements Elicitation

🔄 SDLC 📋 SE Project

The process of seeking and capturing the requirements of a system from users and stakeholders through interviews, surveys, and observation.

↑ Back to Top
S

SAST (Static Application Security Testing)

🔒 Secure Architecture ⚙️ Software Automation

Security testing conducted on source code, bytecode, or binaries without executing the application (white-box testing). Identifies vulnerabilities such as insecure functions, hardcoded credentials, and injection flaws early in the SDLC.

See also: DAST

Scrum

🔄 SDLC 📋 SE Project

A lightweight Agile framework organising development into fixed-length iterations called Sprints (typically 1–4 weeks). Scrum defines roles (Product Owner, Scrum Master, Developers) and ceremonies (Daily Stand-up, Sprint Review, Retrospective).

See also: Agile, Backlog

SDLC (Software Development Life Cycle)

🔄 SDLC 📋 SE Project

The structured set of phases used to plan, design, develop, test, and deliver software. NESA defines four key phases: Defining & Understanding → Planning & Designing → Producing & Implementing → Testing & Evaluating.

SQL Injection

🔒 Secure Architecture

A code injection attack where malicious SQL statements are inserted into an input field and executed by the database. One of the most prevalent web vulnerabilities — prevented via parameterised queries and input validation.

See also: DAST, SAST

Structure Chart

🔄 SDLC 🧩 OOP

A hierarchical tool that shows how a system is divided into modules, the functions within those modules, and how they relate. Used extensively in the Design phase.

Stakeholder

🔄 SDLC 📋 SE Project

Any individual or organization with an interest in a project. In software, this includes users, clients, developers, and project managers.

Syntax Error

📐 Programming Fundamentals

An error in the syntax of a coding or programming language. These errors prevent the code from compiling or running and are usually flagged by the IDE.

See also: Logic Error
↑ Back to Top
U

UML (Unified Modeling Language)

🔄 SDLC 🧩 OOP 📋 SE Project

A standardised visual language for specifying, visualising, constructing, and documenting software systems. HSC-relevant diagrams include Class Diagrams, Use Case Diagrams, Sequence Diagrams, and Activity Diagrams.

Unit Testing

🔄 SDLC ⚙️ Software Automation

The practice of testing individual units (functions, methods, classes) in isolation to verify they behave correctly. Unit tests are the foundation of automated test suites and CI/CD pipelines.

See also: CI/CD

User Acceptance Testing (UAT)

🔄 SDLC 📋 SE Project

The final phase of testing where actual users test the software in real-world scenarios to ensure it meets the requirements and is ready for production.

Use Case Diagram

🔄 SDLC 📋 SE Project

A UML diagram representing the interactions between users (actors) and the system to achieve specific goals (use cases). Helps in defining the system's scope and requirements.

↑ Back to Top
W

WAGILE

🔄 SDLC

A portmanteau of Waterfall and Agile, describing a dysfunctional hybrid where teams adopt Agile ceremonies (stand-ups, sprints) without abandoning heavyweight upfront documentation and rigid change-control from Waterfall. Results in the drawbacks of both models.

See also: Agile, SDLC

White-Box Testing

🔄 SDLC ⚙️ Software Automation

A testing technique in which the tester has full knowledge of internal code structure, logic, and data flows. Used to design tests covering all branches, paths, and conditions — maximising code-coverage metrics.

See also: Black-Box Testing, SAST
