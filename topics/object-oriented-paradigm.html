What is Object-Oriented Programming?

Object-Oriented Programming (OOP)
A programming paradigm that organises software as a collection of objects — each with its own data (attributes) and behaviour (methods) — that interact with each other.

The Four Pillars of OOP

🔒

Encapsulation

Bundling data and methods together; hiding internal details from outside access.

🌳

Inheritance

A class can derive properties and behaviour from a parent class, promoting code reuse.

🎭

Polymorphism

The same interface can be used for different underlying object types. "Many forms".

🎭

Abstraction

Hiding complex implementation details and exposing only what is necessary. "Need to know".

OOP vs Procedural Programming

AspectProceduralObject-Oriented
OrganisationFunctions that operate on dataObjects that combine data and behaviour
ReuseFunction callsInheritance and composition
Data accessGlobal or passed as parametersControlled via access modifiers
ScalabilityHarder to manage in large programsBetter for large, complex systems
ExamplesC, PascalPython, Java, C++, C#

Classes & Objects

📐

Class

A blueprint or template for creating objects. Defines what attributes (data) and methods (behaviour) objects of that type will have. Like a cookie cutter.

🍪

Object (Instance)

A specific instance created from a class. Each object has its own attribute values. Like a cookie made from the cutter.

Python — Defining and Using a Class 
class BankAccount:
    """Represents a simple bank account."""

    # Class attribute (shared by all instances)
    interest_rate = 0.03

    def __init__(self, owner, balance=0):
        # Instance attributes (unique to each object)
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        """Add funds to the account."""
        if amount > 0:
            self.balance += amount
        return self.balance

    def withdraw(self, amount):
        """Remove funds if sufficient balance exists."""
        if amount > self.balance:
            raise ValueError("Insufficient funds")
        self.balance -= amount
        return self.balance

    def __str__(self):
        return f"Account({self.owner}, ${self.balance:.2f})"


# Creating objects (instantiation)
alice_account = BankAccount("Alice", 1000)
bob_account   = BankAccount("Bob")

# Using methods
alice_account.deposit(500)
alice_account.withdraw(200)
print(alice_account)          # Account(Alice, $1300.00)
print(alice_account.balance)  # 1300.0

__init__ is a special (dunder) method called automatically when an object is created. It initialises the object's attributes. self refers to the current instance.

Encapsulation

Encapsulation
Bundling data (attributes) and methods together within a class, and restricting direct access to some of the object's components. The internal state is accessed and modified only through defined methods (getters and setters).

Access Modifiers in Python

ConventionSyntaxAccess Level
Publicself.nameAccessible from anywhere
Protectedself._nameConvention: internal use (1 underscore)
Privateself.__nameName-mangled; not directly accessible (2 underscores)
ℹ️
Python's Approach

Python uses naming conventions rather than strict access modifiers. The double-underscore __attr triggers name-mangling to _ClassName__attr, making accidental access harder but not impossible. "We're all consenting adults here."

Getters and Setters

Python — Encapsulation with Properties 
class Student:
    def __init__(self, name, age):
        self.name = name
        self.__age = age  # private attribute

    @property
    def age(self):
        """Getter: returns the age."""
        return self.__age

    @age.setter
    def age(self, value):
        """Setter: validates before setting age."""
        if not isinstance(value, int) or value < 0:
            raise ValueError("Age must be a positive integer")
        self.__age = value

student = Student("Alice", 17)
print(student.age)    # 17 (via getter)
student.age = 18     # via setter — validates input
# student.age = -5  → raises ValueError

Benefits of Encapsulation

  • Protects object's internal state from unintended modification
  • Allows validation logic in setters
  • Implementation can change without affecting external code
  • Reduces complexity by hiding unnecessary details

Inheritance

Inheritance
A mechanism where a child class (subclass) derives attributes and methods from a parent class (superclass). Promotes code reuse and establishes an "is-a" relationship.
Python — Inheritance Example 
class Animal:
    """Base class for all animals."""

    def __init__(self, name, species):
        self.name = name
        self.species = species

    def eat(self):
        print(f"{self.name} is eating")

    def speak(self):
        print("...")  # generic — overridden by subclasses

    def __str__(self):
        return f"{self.name} ({self.species})"


class Dog(Animal):  # Dog inherits from Animal
    def __init__(self, name, breed):
        super().__init__(name, "Canis lupus familiaris")
        self.breed = breed  # additional attribute

    def speak(self):       # override parent method
        print(f"{self.name} says: Woof!")

    def fetch(self):       # new method unique to Dog
        print(f"{self.name} fetches the ball!")


class Cat(Animal):
    def speak(self):
        print(f"{self.name} says: Meow!")


# Using the classes
dog = Dog("Rex", "Labrador")
cat = Cat("Whiskers", "Felis catus")

dog.eat()     # Inherited: Rex is eating
dog.speak()   # Overridden: Rex says: Woof!
dog.fetch()   # Dog-specific: Rex fetches the ball!
cat.speak()   # Whiskers says: Meow!

Types of Inheritance

TypeDescriptionPython Support
SingleOne subclass, one superclass (A → B)✅
Multi-levelChain of inheritance (A → B → C)✅
MultipleOne subclass inherits from multiple superclasses✅ (with MRO)
HierarchicalMultiple subclasses from one superclass (A → B, A → C)✅

super()

super() returns the parent class and allows calling its methods. Essential when overriding __init__ to ensure the parent's initialisation also runs.

Polymorphism

Polymorphism
"Many forms" — the ability for different object types to be treated as instances of a common interface, each responding to the same method call in their own way.

Method Overriding (Runtime Polymorphism)

Python — Polymorphism via method overriding 
class Shape:
    def area(self):
        raise NotImplementedError("Subclasses must implement area()")

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    def area(self):
        return 3.14159 * self.radius ** 2

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    def area(self):
        return self.width * self.height

class Triangle(Shape):
    def __init__(self, base, height):
        self.base = base
        self.height = height
    def area(self):
        return 0.5 * self.base * self.height


# Polymorphism: same method call, different results
shapes = [Circle(5), Rectangle(4, 6), Triangle(3, 8)]
for shape in shapes:
    print(f"{type(shape).__name__}: area = {shape.area():.2f}")
# Circle: area = 78.54
# Rectangle: area = 24.00
# Triangle: area = 12.00

Duck Typing

In Python, polymorphism is achieved through duck typing: "If it walks like a duck and quacks like a duck, it's a duck." Python doesn't require formal inheritance — if an object has the required method, it will work.

Operator Overloading

Python — Operator Overloading 
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):       # overload +
        return Vector(self.x + other.x, self.y + other.y)

    def __str__(self):
        return f"Vector({self.x}, {self.y})"

v1 = Vector(1, 2)
v2 = Vector(3, 4)
print(v1 + v2)  # Vector(4, 6)

Abstraction

Abstraction
Hiding complex implementation details and showing only the necessary features of an object. Users interact with objects through a simplified interface without needing to understand how things work internally.

Abstract Classes in Python

An abstract class cannot be instantiated directly. It defines a contract — methods that all subclasses must implement. Python uses the abc module for abstract base classes.

Python — Abstract Base Class 
from abc import ABC, abstractmethod

class Vehicle(ABC):  # Abstract base class
    def __init__(self, make, model):
        self.make = make
        self.model = model

    @abstractmethod
    def start_engine(self):
        """All vehicles must implement this."""
        pass  # subclasses provide implementation

    @abstractmethod
    def fuel_type(self):
        pass

    def describe(self):  # Concrete method (shared)
        return f"{self.make} {self.model}"


class Car(Vehicle):
    def start_engine(self):
        print("Car engine: Vroom!")
    def fuel_type(self):
        return "Petrol"

class ElectricCar(Vehicle):
    def start_engine(self):
        print("Electric motor: Whoosh!")
    def fuel_type(self):
        return "Electric"

# v = Vehicle("X", "Y")  ← TypeError: Can't instantiate abstract class
car = Car("Toyota", "Camry")
car.start_engine()  # Car engine: Vroom!

Exception Handling

Exception
An error that occurs during program execution that disrupts the normal flow of instructions. Examples: ZeroDivisionError, FileNotFoundError, TypeError, ValueError, IndexError.
Python — Exception Handling 
try:
    numerator = int(input("Enter numerator: "))
    denominator = int(input("Enter denominator: "))
    result = numerator / denominator
    print(f"Result: {result}")

except ZeroDivisionError:
    print("Error: Cannot divide by zero!")

except ValueError:
    print("Error: Please enter valid integers!")

except Exception as e:
    print(f"Unexpected error: {e}")

else:
    # Runs only if no exception occurred
    print("Calculation successful!")

finally:
    # Always runs (cleanup code)
    print("Division operation completed.")

Custom Exceptions

Python — Custom Exception Class 
class InsufficientFundsError(Exception):
    """Raised when a bank withdrawal exceeds the balance."""
    def __init__(self, amount, balance):
        self.amount = amount
        self.balance = balance
        super().__init__(
            f"Cannot withdraw ${amount:.2f}: balance is ${balance:.2f}"
        )

# Using the custom exception
def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientFundsError(amount, balance)
    return balance - amount

try:
    new_balance = withdraw(100, 150)
except InsufficientFundsError as e:
    print(e)  # Cannot withdraw $150.00: balance is $100.00
✅
OOP Key Takeaways
  • The four pillars: Encapsulation, Inheritance, Polymorphism, Abstraction
  • A class is a blueprint; an object is an instance of a class
  • __init__ initialises object attributes; self refers to the current instance
  • Inheritance enables "is-a" relationships and code reuse; use super() to call parent methods
  • Polymorphism: different objects respond to the same method in their own way
  • UML class diagrams show + public, - private, # protected members
  • Always handle exceptions with try/except — never let programs crash unexpectedly
