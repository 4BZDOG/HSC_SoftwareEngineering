📊 System and Data Modelling Tools

📊 Data Flow Diagrams (DFD)

Used to represent the flow of data through an information system.

⭕ CircleProcess — uses input to generate output.
🗃️ Open RectangleData Store — file or non-computer storage.
🏢 RectangleExternal Entity — source or sink of data.
↩️ Curved ArrowData Flow — movement of data between elements.
Level 0 DFD: Represents an overview of the entire system; does not show data stores or internal processes.

🌐 Structure Charts

Represent a system by showing separate subroutines and their relationships.

🔘 Empty Circle ArrowData Movement — passes parameters between modules.
⚫ Filled Circle ArrowControl Flag — passes status variables between modules.
🔷 DiamondDecision — indicates optional or conditional execution.
🔁 Curved ArrowRepetition — indicates a looping structure.

📊 Data Dictionary

Comprehensive description of each variable, including: Name, Data Type, Format, Size (bytes/display), Description, Example, and Validation.

📋 Project Management Tools

📈 Gantt Charts

Displays component tasks on an estimated timeline, showing dependencies and milestones.

📝 Process Diaries / Log Books

Regular interval documentation of progress, tasks achieved, stumbling blocks, and reflective comments.

⌨️ Programming Paradigms

🧩 Object-Oriented

Classes, objects, attributes, methods, inheritance, polymorphism, and encapsulation.

📝 Logic

Facts, rules, and displaying solutions/rules used by the system.

📊 Imperative

Control structures, assignment statements, expressions, and subroutines.

⚙️ Functional

Recursion, first-class objects, abstraction, and encapsulation.

📝 Algorithms and Logic

💻 Pseudocode Standards

Keywords in CAPITALS, structural elements in pairs (BEGIN/END, IF/ENDIF), and indentation for control structures.

🌊 Flowchart Symbols

🔵 OvalTerminator — marks the Start or End of a flowchart.
📐 ParallelogramInput/Output — represents data entering or leaving the system.
▭ RectangleProcess — a computation or action step.
🔶 DiamondDecision — a branching condition (Yes/No or True/False).
🗂️ Double-sided RectangleSubprogram — a call to a separate subroutine or function.

🕹️ Control Structures

StructurePseudocode Example
Binary SelectionIF condition THEN ... ELSE ... ENDIF
Multi-way SelectionCASEWHERE expression ... OTHERWISE ... END CASE
Pre-test LoopWHILE condition is true ... ENDWHILE
Post-test LoopREPEAT ... UNTIL condition is true
Counted LoopFOR variable = start TO finish STEP increment ... NEXT variable

🗄️ Relational Databases (SQL)

Standard SQL Syntax 
SELECT field(s)
FROM table(s)
WHERE search criteria
GROUP BY field(s)
ORDER BY field(s) ASC/DESC

Includes functions like SUM, AVG, COUNT, MAX, MIN and operators like CONTAINS, EQUALS, AND, OR, NOT.

🌐 Web and Machine Learning

🌐 Programming for the Web

  • Front-end frameworks (React, Vue, etc.)
  • Cross-site scripting (XSS) prevention and sanitisation.
  • CSS Syntax (Selectors, Properties, Values).

📋 Machine Learning (MLOps)

The three stages of MLOps:

  1. Design: Problem definition, metrics, data research.
  2. Model Development: Wrangling, engineering, training, validation.
  3. Operations: Deployment, monitoring, performance.

⚙️ Methods for Testing and Representation

⚙️ Testing Methods

🔧 FunctionalVerifies the system operates according to its functional requirements and produces expected outputs.
✅ AcceptanceDetermines whether the system meets stakeholder requirements and is ready for deployment.
📡 Live DataTests using real-world production data to identify issues in a live environment.
🎲 Simulated DataUses artificially generated data that mirrors expected real-world scenarios.
🧑‍💻 BetaReleases the product to a limited group of real users to identify bugs before full release.
📦 VolumeEvaluates system performance under large amounts of data to test scalability.

🧠 Character Representation

Characters are represented using ASCII or Unicode. Developers use binary internal formats for functions like case conversion or simple encryption.
