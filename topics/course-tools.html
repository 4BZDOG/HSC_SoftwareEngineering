📊 System and Data Modelling Tools

📊 Data Flow Diagrams (DFD)

Used to represent the flow of data through an information system.

⭕ CircleProcess — uses input to generate output.
🗃️ Open RectangleData Store — file or non-computer storage.
🏢 RectangleExternal Entity — source or sink of data.
↩️ Curved ArrowData Flow — movement of data between elements.
Level 0 DFD: Represents an overview of the entire system; does not show data stores or internal processes.

🌐 Structure Charts

Represent a system by showing separate subroutines and their relationships.

🔘 Empty Circle ArrowData Movement — passes parameters between modules.
⚫ Filled Circle ArrowControl Flag — passes status variables between modules.
🔷 DiamondDecision — indicates optional or conditional execution.
🔁 Curved ArrowRepetition — indicates a looping structure.

📊 Data Dictionary

Comprehensive description of each variable, including: Name, Data Type, Format, Size (bytes/display), Description, Example, and Validation.

🧩 Class Diagrams

Used in object-oriented design to show classes, their attributes, methods, and relationships. Each class box is divided into three sections: the class name, attributes, and methods.

— (solid line)Association — a relationship between two classes.
▷ (open arrow)Inheritance — a subclass extends a superclass.
◇ (open diamond)Aggregation — a "has-a" relationship (weak ownership).
◆ (filled diamond)Composition — a strong "contains" relationship (lifecycle dependency).

🖼️ Storyboard

A visual sequence of screen layouts or interface states that communicates the intended user experience and navigation flow through a system. Used in the design phase to plan user interfaces before development begins.

🌳 Decision Trees

A tree-based diagram that maps out decision logic, showing possible conditions and the resulting outcomes for each branch. Each internal node represents a condition; each leaf node represents an outcome.

📋 Project Management Tools

📈 Gantt Charts

Displays component tasks on an estimated timeline, showing dependencies and milestones.

📝 Process Diaries / Log Books

Regular interval documentation of progress, tasks achieved, stumbling blocks, and reflective comments.

⌨️ Programming Paradigms

🧩 Object-Oriented

Classes, objects, attributes, methods, inheritance, polymorphism, and encapsulation.

📝 Logic

Facts, rules, and displaying solutions/rules used by the system.

📊 Imperative

Control structures, assignment statements, expressions, and subroutines.

⚙️ Functional

Recursion, first-class objects, abstraction, and encapsulation.

📝 Algorithms

💻 Pseudocode

Keywords in CAPITALS, structural elements in pairs (BEGIN/END, IF/ENDIF), and indentation for control structures.

🌊 Flowcharts

🔵 OvalTerminator — marks the Start or End of a flowchart.
📐 ParallelogramInput/Output — represents data entering or leaving the system.
▭ RectangleProcess — a computation or action step.
🔶 DiamondDecision — a branching condition (Yes/No or True/False).
🗂️ Double-sided RectangleSubprogram — a call to a separate subroutine or function.

🕹️ Control Structures

Sequence

Statements execute one after another in the order they are written.

Pseudocode
BEGIN
  statement1
  statement2
  statement3
END

Selection

Binary Selection

Pseudocode
IF condition THEN
  statements
ELSE
  statements
ENDIF

Multi-way Selection

Pseudocode
CASEWHERE expression equals
  value1: statements
  value2: statements
  OTHERWISE: statements
END CASE

Nested IF

Pseudocode
IF condition1 THEN
  IF condition2 THEN
    statements
  ELSE
    statements
  ENDIF
ELSE
  statements
ENDIF

Repetition

Pre-test

Pseudocode
WHILE condition is true
  statements
ENDWHILE

Post-test

Pseudocode
REPEAT
  statements
UNTIL condition is true

FOR / NEXT

Pseudocode
FOR variable = start TO finish STEP increment
  statements
NEXT variable

🔧 Subroutines

Using a subroutine with one parameter

Pseudocode
SUBROUTINE name(param)
  statements
END SUBROUTINE

// Calling the subroutine:
name(value)

Using a subroutine with multiple parameters

Pseudocode
SUBROUTINE name(param1, param2)
  statements
END SUBROUTINE

// Calling the subroutine:
name(value1, value2)

Passing a value back from a function

Pseudocode
FUNCTION name(param)
  statements
  RETURN value
END FUNCTION

// Calling the function:
result ← name(value)

🗄️ Relational Databases

SQL

Standard SQL Syntax 
SELECT field(s)
FROM table(s)
WHERE search criteria
GROUP BY field(s)
ORDER BY field(s) ASC/DESC

Includes functions like SUM, AVG, COUNT, MAX, MIN and operators like CONTAINS, EQUALS, AND, OR, NOT.

Object-Relational Mapping (ORM)

A technique that maps object-oriented class structures to relational database tables, allowing developers to interact with a database using objects rather than raw SQL. Each class maps to a table; each attribute maps to a column; each object instance maps to a row.

Class → TableEach class in the model corresponds to a database table.
Attribute → ColumnEach attribute of the class becomes a column in the table.
Object → RowEach instance of the class is stored as a row in the table.

🌐 Programming for the Web

Front-end web development frameworks

Libraries and frameworks such as React, Vue, and Angular that provide reusable components and structure for building user interfaces in the browser.

Cross-site scripting (XSS)

A security vulnerability where malicious scripts are injected into web pages viewed by other users. Mitigated by sanitising user input and escaping output before rendering it in the browser.

Cascading Style Sheets (CSS)

A stylesheet language used to describe the presentation of HTML documents. CSS rules use a selector, property, and value syntax.

CSS Syntax
selector {
  property: value;
}
SelectorTargets the HTML element(s) to style (e.g. p, .class, #id).
PropertyThe style attribute to change (e.g. color, font-size, margin).
ValueThe setting to apply to the property (e.g. red, 16px, auto).

🤖 Machine Learning

Machine learning automation through DevOps

The integration of machine learning workflows with DevOps practices (MLOps). The three stages:

  1. Design: Problem definition, metrics, data research.
  2. Model Development: Wrangling, engineering, training, validation.
  3. Operations: Deployment, monitoring, performance.

Regression algorithms

Supervised learning algorithms used to predict a continuous output value based on input features.

Linear RegressionModels the relationship between variables using a straight line: y = mx + b.
Logistic RegressionUsed for classification tasks; outputs a probability between 0 and 1.
Polynomial RegressionExtends linear regression to model curved (non-linear) relationships.

Neural Networks

Computational models inspired by biological neural networks, composed of layers of interconnected nodes (neurons).

Training cycle

Forward passInput data is passed through the network to generate a prediction.
Loss calculationThe error between the prediction and the expected output is measured.
BackpropagationThe error is propagated backwards through the network to update weights.
Weight updateWeights are adjusted using an optimiser (e.g. gradient descent) to reduce loss.

Execution cycle

InferenceThe trained model receives new input and generates a prediction (forward pass only).
DeploymentThe model is integrated into an application or service for real-world use.

🧪 Methods for Testing a System

🔧 FunctionalVerifies the system operates according to its functional requirements and produces expected outputs.
✅ AcceptanceDetermines whether the system meets stakeholder requirements and is ready for deployment.
📡 Live DataTests using real-world production data to identify issues in a live environment.
🎲 Simulated DataUses artificially generated data that mirrors expected real-world scenarios.
🧑‍💻 BetaReleases the product to a limited group of real users to identify bugs before full release.
📦 VolumeEvaluates system performance under large amounts of data to test scalability.

🔣 Character Representation

Characters are represented using ASCII or Unicode. Developers use binary internal formats for functions like case conversion or simple encryption.

ASCII7-bit encoding representing 128 characters (letters, digits, symbols, control codes).
UnicodeInternational standard encoding supporting over 140,000 characters across most writing systems.
UTF-8Variable-width encoding for Unicode; backwards-compatible with ASCII.

🐍 Programming with Python

Python is the primary language used in the HSC Software Engineering course. Key syntax and conventions:

Python — Core Syntax
# Variables and assignment
name = "Alice"
age = 17

# Input / Output
name = input("Enter name: ")
print("Hello,", name)

# Selection
if age >= 18:
    print("Adult")
elif age >= 13:
    print("Teenager")
else:
    print("Child")

# Iteration
for i in range(1, 6):
    print(i)

count = 0
while count < 5:
    count += 1

# Functions
def greet(name):
    return "Hello, " + name

# Lists
scores = [85, 92, 78]
scores.append(95)

# File I/O
with open("data.txt", "r") as f:
    content = f.read()
