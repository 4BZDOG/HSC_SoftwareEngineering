```
1. Source Code (.mmd files in source-code/)
           ↓
2. Validate (https://mermaid.live)
           ↓
3. Generate SVG (mmdc or online tool)
           ↓
4. Optimize (reduce file size)
           ↓
5. Store (diagrams/generated/[topic]/)
           ↓
6. Reference in HTML (with alt text)
           ↓
7. Test (browser, mobile, accessibility)
           ↓
8. Version Control (commit source + SVG)
```

## 🎨 Diagram Styling Guide

All diagrams use consistent colors and styles:

### Color Scheme
- 🟦 **Blue (#E3F2FD):** Input, Start, Sources
- 🟪 **Purple (#F3E5F5):** Processing, Logic, Transformations
- 🟩 **Green (#E8F5E9):** Output, Result, Destinations
- 🟧 **Orange (#FFF3E0):** Intermediate steps, Cache, Storage
- 🟥 **Red (#FF6B6B):** Errors, Issues, Warnings
- 🟨 **Yellow (#FFC107):** Decisions, Checkpoints

### Typography
- **Titles:** 18-24px, bold, primary color
- **Labels:** 14px, medium weight
- **Descriptions:** 12px, secondary color
- **Icons:** Emoji for quick visual reference

### Layout
- **Max width:** 800-1000px (responsive)
- **Padding:** 1-2rem around diagram
- **Border radius:** Consistent with site theme
- **Shadow:** Subtle elevation effect

## 📝 Contributing New Diagrams

To add a new diagram:

1. **Create source file:**
   ```bash
   diagrams/source-code/[category]/[name].mmd
   ```

2. **Write Mermaid code:**
   - Reference [QUICK_REFERENCE.md](QUICK_REFERENCE.md)
   - Follow styling guide above
   - Test at https://mermaid.live

3. **Generate SVG:**
   ```bash
   mmdc -i diagram.mmd -o ../generated/[topic]/diagram.svg
   ```

4. **Document in README.md:**
   - Add to appropriate section
   - Include scenario, purpose, and Mermaid code
   - Link to generated SVG path

5. **Update INDEX.md:**
   - Add to diagram listing
   - Mark status as ✅ Ready or ⏳ Pending

6. **Commit changes:**
   ```bash
   git add diagrams/source-code/[category]/[name].mmd
   git add diagrams/generated/[topic]/[name].svg
   git add diagrams/README.md
   git add diagrams/INDEX.md
   ```

## 🐛 Troubleshooting

### Diagram doesn't render
- Check Mermaid syntax validity
- Ensure script tag: `<script src="mermaid.min.js"></script>`
- Verify `mermaid.contentLoaded()` is called

### SVG file too large
- Simplify diagram (fewer nodes)
- Use `mmdc -w 800 -H 600` to constrain size
- Optimize SVG: `svgo diagram.svg`

### Diagram unclear
- Add descriptive labels
- Use colors consistently
- Break into multiple smaller diagrams
- Include legend or explanation
